Задача

Сделать приложение для учета калорий

Вводные

Файл kulinar.json - данные о продуктах - названия и калоррийность (на 100 грамм продукта)

Файл backend.php - примитивный "бекэнд", роль его заключаетс в том, что любые данные, которые вы пришлете в POST запросе, он вытащит, сделает из этого объект, добавит метку времени в поле timestamp и приклеит объект к json массиву в файле eater.json

Механики

Предполагается, что пользователь в левой панели видит картинки продуктов, перетаскивает их в правую панель, попутно указывая сколько граммов продукта использует. Получившееся "блюдо" пользователь "съедает". Мы созраняем данные о съеденном

Задание со звездочкой - на основании сохраненных данных сделать "витрину" - страничку где будет информация о ходе "диеты" пользователя

Часто задаваемые вопросы

Картинок нет, это ожидаемо. Ищите картинки на Яндекс.Картинках (будем считать, что для учебного проекта можно использовать любые изображения из интернета), unsplash.com, flaticon.com или любом другом сервисе. Надеюсь, картинки будут подобраны с любовью

Придумали свою реализацию интерфейса? Если это не ломает общую идею учета калорий и это действительно удобно/классно - вперед. Не забудьте только описать, что и почему вы делали.

Хотите больше данных в исходном файле, потому что придумали крутую фичу? Без проблем. Опять же, опишите зачем вам потребовались эти данные текстом
